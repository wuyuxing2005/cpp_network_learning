# 实现简单的通信

## 1.主机字节序和网络字节序：

主机字节序指的就是小端字节序存储，网络字节序指的是大端字节序存储

- 规定：
发送端需要将数据转主机字节序化为网络字节序，接收端要将网络字节序转化为主机字节序
原因就是不同设备的存储方式不同，人为规定可以实现不同字节序的设备进行交互。

- 转化函数(就列举常用的)：
  - ip的转化：
    - inet_pton: 十分制ip转化为网络字节序
    - inet_ntop: 网络字节序转化为十分制ip
  - 端口号的转化：
    - htons：主机端口号转化为网络字节序
    - ntohs：网络字节序转化为主机端口号

## 2.socket与socketaddress的创建

- socket 包括监听sokcet与连接socket，
  - 在服务器端所建立的并绑定socket_address的是监听socket，其作用就是监听该socket上是否有请求连接的请求到达。
  - 在客户端所建立的是连接socket，它需要连接你想要连接的服务器的监听socket_address。
- socket指的就是一个端口号和ip地址的二元对，即（port，ip）
- socket地址：
  - 即该socket的具体内容
    - 结构体：struct sockaddr_in **(IPV4)**
      - sin_family: 地址族，一般为AF_INET
      - sin_port: 端口号，使用htons函数转化为网络字节序
      - sin_addr: ip地址，使用inet_pton函数转化为网络字节序
    - 其中sin_addr又是一个结构体：
      - struct in_addr
        - s_addr: 存储ip地址，使用inet_pton函数转化为网络字节序
一般的创建socket地址的代码示例为：

```cpp
    sockaddr_in sock_address{};
    sock_address.sin_family = AF_INET;//表示是IPV4
    sock_address.sin_port = htons(9999);//设置端口号
    sock_address.sin_addr.s_addr= shifentonet("127.0.0.1")//设置ip地址

```

- socket创建：
  - socket实际上仍然是文件，存在唯一的文件描述符。

```cpp
    int socket_fd=socket(AF_INET,SOCK_STREAM,0);//SOCK_STREAM表示使用TCP，SOCK_STREAM表示使用UDP
    //socket_fd即为新创建的套接字的文件描述符      
```

## 3. socket连接与通信过程

- socket与socket_address使用bind绑定
  - 只需要在服务器端的监听socket进行绑定

    ```cpp
     bind(sokcet_fd,(sockaddr *)&socket_address,sizeof(socket_address));
    ```

    - 其中socket_address要转化为sockaddr类型

- **连接socket**的connect
```cpp
connect(socket_fd,(sockaddr*)&sc_address,sizeof(sc_address));
```
此处的connect的含义是让当前的连接socket向监听socket对应的address发送连接申请,词后需要等待server端的监听socket accpet
  
- **监听socke**t的listen：
  ```cpp
  listen(socket_fd, SOMAXCONN);//SOMAXCONN指的是内核允许的“已完成连接队列”的最大长度
  //即等待accpet的队列的最大长度
  ```
- **监听socket**的accept：
  监听到连接之后，可以对监听队列进行accpet；
```cpp
int client_fd=accept(fd, (sockaddr *)&sc_address, &sizeof(sc_address));
```
此处的返回值是accpet到的连接socket的fd。之后client端和server端将通过该client_fd进行通讯。
